# 解説レイアウト機能 実装指示書（既存テック雑学プロジェクトに追加）

## 目的
ゆっくり解説系で最も多用される「画像を縮小して、空いたスペースに解説テキストを段階表示する」レイアウトを実装する。
1枚の画像にトランジションで小細工するのではなく、**画像の表示サイズを変えてテキスト領域を作り、ナレーションに合わせて要点を積んでいく**動き。これが解説動画の土台レイアウトであり、矢印や数値強調もこの中に乗せる。

（参考イメージ：画像が全体表示 → 縮小して左に寄り、右側に「50万時間 → 無給労働 → もったいない」と矢印付きで順に現れる、という動き）

## 大前提（厳守）
- **後方互換を最優先**：この演出を指定していない既存の台本・既存動画が、これまで通り問題なく書き出せること。新レイアウトは「任意で追加できる」拡張にする。
- **データと描画を分離**：台本側の「演出の定義（データ）」と、Remotion側の「描画ロジック」を明確に分ける。
- **外部の有料API・有料サービスは使わない。**
- **UIはフラット・ミニマル**（参考イメージの雰囲気。装飾過多にしない）。
- **まず最小で動かす。** 凝った同期・装飾は後回し。

## 演出の出どころについて（案A方針）
当面は、台本生成AIが台本と一緒にこの演出指定（後述のフィールド）も出力する前提で実装する。
つまりこの指示書は「**台本にこのフィールドが入っている前提で、それを描画するRemotion側の実装**」を対象とする。
（演出指定を台本生成AIに出させるプロンプト改修は別途行う。演出の質が不足する場合は、将来的に演出専用AIを挟む案Bに格上げするが、今は案Aで進める。）

## 機能仕様

### 1. 台本JSONスキーマの拡張
既存スキーマを壊さず、以下の任意フィールドを追加する。

- `chapters` の各章に、任意で `panel` オブジェクトを持てるようにする：
  ```
  "panel": {
    "image": "（この章で見せるメイン画像。既存の image_cuts の主画像を流用してもよい）",
    "items": [
      { "text": "50万時間" },
      { "text": "無給労働", "arrow_from_prev": true },
      { "text": "もったいない", "arrow_from_prev": true }
    ]
  }
  ```
  - `items` は解説テキストの配列（2〜4個程度）。
  - `arrow_from_prev: true` のとき、直前の項目から矢印でつなぐ（簡易フロー表現を兼ねる）。

- `script` の各発言に、任意で次のフィールドを持てるようにする：
  - `"panel_event": "shrink"` … この発言の再生時に、画像を縮小して定位置へ移動し、テキスト領域を開く。
  - `"panel_item": n`（整数）… この発言の再生時に、`items[n]` を出現させる。

- いずれも任意。`panel` が無い章はこれまで通り（image_cuts による画像表示）。

### 2. 表示挙動
- `panel` を持つ章では、章の冒頭で画像を全体表示する。
- `panel_event: "shrink"` の発言で、画像を縮小して定位置（例：左寄せ）へ移動し、空いた側にテキスト領域を開く。
- `panel_item: n` の発言で、`items[n]` を出現させる。`arrow_from_prev` があれば前項目から矢印を引く。到達前の項目は表示しない、出た項目は残す（積み上がっていく）。
- これらの指定が無い場合のフォールバック：画像は全体表示のまま、章の経過時間で items を等間隔に出現させる。
- 既存の `cut` / `effect` / `image_cuts` と併存できること。

### 3. 見た目
- 参考イメージの雰囲気：縮小した画像＋大きめの解説テキスト＋必要なら矢印。
- フラット・ミニマル。アクセントは控えめに。

### 4. このフェーズの完了条件
- `panel` を持つサンプル台本を1本用意し、ナレーション進行に合わせて「画像縮小 → テキストが順に出現（矢印含む）」が再生され、最後まで動画として書き出せること。
- `panel` を持たない既存台本が、これまで通り問題なく書き出せること（回帰確認）。

---

## 進め方
1. 既存テック雑学プロジェクトに、この `panel` レイアウト機能を追加実装する（今回はこれのみ）。
2. 雑学の台本1本にサンプルの `panel` を手で書き込み、動作と見た目を確認する。
3. 既存台本が壊れていないか回帰確認する。
4. 問題なければ報告。次段階として「台本生成AIが `panel` を自動出力する」プロンプト改修に進む。
5. その後、別テーマへの展開を行う。
