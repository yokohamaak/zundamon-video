# V2編集機能の互換性棚卸し（2026-07-16）

## 結論

**要修正。** V2の舞台モデル・レンダラーは旧方式とは別の契約で実装されている一方、編集画面は旧エディタの共通操作をそのまま見せている。結果として、次の三種類が混在している。

1. V2で正しく使える操作
2. V2では画面から到達できないが、データ形式・レンダラーは対応済みの操作
3. V2で実行すると旧フィールドを書き込み、保存validatorに拒否される、または状態遷移を壊し得る操作

「旧と同じ操作があるか」だけでなく、**UI操作 → 保存JSON → 音声生成 → プレビュー/Remotion描画**まで到達するかを静的に照合した。これは実装は行わない棚卸しである。レンダリング済み機能の監査は [v2-stage-implementation-audit-2026-07-16.md](v2-stage-implementation-audit-2026-07-16.md) を参照。

## 修正後の対応状況（2026-07-16）

この棚卸しを基に、吹き出し文字数の仕様（E-10）を除く編集経路を修正した。

| ID | 状況 | 修正内容 |
| --- | --- | --- |
| E-01 / E-02 | ✅ | V2の追加・引継追加は、有効なsceneと話者個体を使い、`stage`イベントを複製しない。 |
| E-03 / E-06 | ✅ | V2では旧方式の一括編集を表示・実行しない。V2用に意味を定義するまでは旧フィールドを作らせない。 |
| E-04 / E-09 | ✅ | V2セリフ欄にカーソル位置でのターン分割を追加。後半はstageを持たず、通常表示時だけ吹き出しを連結する。自動文分割は従来どおり別機能。 |
| E-05 | ✅ | V2の削除・上下移動は、登場/退場/更新と話者構図の順序が壊れる場合に中止する。保存時のstrict validatorも最終防壁として残す。 |
| E-07 / E-08 | ✅ | `pause`、口パク停止、手動SE、ターン別字幕スタイルをV2詳細パネルで編集できるようにした。pause変更時は音声再生成を要求する。 |
| E-10 | 保留 | 吹き出しの文字数・横幅・改行・連結の意味を再定義してから別途対応する。 |

## 根本原因

V2の詳細パネルは `v2SetEditorVisibility()` により旧基本タブを隠す一方（`story_editor.html:7905-7920`）、ターン追加・分割・一括編集・タイムラインなどの**詳細パネル外の共通操作はschema分岐されていない**。V2で許されるフィールドは `stage_schema.py` が厳格に制限しているため、旧操作を通すと「編集できたように見えて保存できない」状態になる。

この問題はレンダラーの不足ではなく、エディタ操作の入口とV2状態遷移の対応表が無かったことによる。

## 修正に着手する順番

1. **P0安全化:** V2時は危険な一括編集を非表示/無効化し、追加・引継追加をV2専用状態遷移にする。保存不能なJSONをUIから作らせない。
2. **編集継続性:** V2版の手動セリフ分割を追加する。分割後の後半turnにstageイベントを複製しないことを仕様化する。
3. **必要な旧操作の回復:** `pause`、`noLipSync`、手動SE、ターン別字幕スタイルを、必要性の確認順にV2 UIへ移植する。
4. **一括編集の再設計:** V2用として、scene / slot配置 / stage構図 / 話者のうち安全に一括適用できるものだけを設計して追加する。旧一括編集の流用は禁止する。
5. **吹き出しは別件として再設計:** 文字数上限、横幅、段分割、連結、字幕帯を混同しない仕様を決めてから対応する。

## 実装前に決めること

1. 新規ターン追加時の既定値: **推奨は「直前turnのsceneとspeakerを引継ぎ、stageイベントは空」**。直前sceneが無い先頭ターンでは、有効sceneと話者instanceを選ばせる必要がある。
2. 手動セリフ分割後の吹き出し: **推奨は後半turnだけ `continueBubble: true`**。ただし字幕帯・特殊表示では連結しない。
3. V2で回復する旧機能の範囲: `pause` / `noLipSync` / 手動SE / ターン別字幕スタイルを全部回復するか、最小の `pause` と `noLipSync` から始めるか。
4. 一括編集: V2で必要な対象を、旧の表情一括・テロップ一括ではなく「slot移動」「stage構図」「話者」などに絞るか。

## 未確認事項

- 本資料はコード読み取りによる静的棚卸しであり、各操作をブラウザ実機で全件クリックした結果ではない。
- E-01〜E-06は、ソース上で不正なV2フィールドまたは状態を作る経路を確認済み。修正時はV2 fixtureで「追加 → 保存」「引継追加 → 保存」「複数選択 → UI無効」を自動/手動で検証する。
- BGM、Overlay、モブ特殊表示、前景、camera motionは、前回の実装監査でV2 renderer対応済みと確認している。本資料では再監査せず、編集入口の有無だけを扱う。
