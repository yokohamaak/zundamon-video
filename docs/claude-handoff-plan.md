# Claude Code 引き継ぎ計画

Codex不在時に、ユーザーとClaude Codeだけで安全に開発を進めるための手順。
基準コミットは `bc653c2`。Phase 1-3と旧UIデッドコード撤去まで完了済み。

## 現在地

- editorモデルの正: `assets` / `imageCues` / `visualSegments`
- 移行済み判定: `editorModelAuthority === "editor"`
- 移行: プロジェクト単位の手動操作のみ。自動一括移行は禁止
- legacyの読込・編集・meta生成と、editorから既存metaへの変換は後方互換として維持
- rollback: 切替前の `.backups/editor-authority-pre-*` を使用。フラグだけを安易に消して戻さない
- Phase 4（Remotion合成面）は未着手

## 共通実行規則

各タスクの冒頭に、以下をClaudeへ渡す。

```text
AGENTS.mdとdocs/claude-handoff-plan.mdを先に読むこと。
今回指定した増分だけ実装し、次の増分へ自動で進まないこと。

実装前に次を列挙してから着手する:
1. 同じデータを読む・追加・移動・削除する全経路
2. 連動データと保存・再読込・meta生成への影響
3. null/空/0/重複/範囲端/章境界/旧データ混在
4. legacy/editor双方への影響

共通処理へ集約し、典型経路だけを個別修正しないこと。
先に失敗ケースを含むテスト名を決め、ローカルテストを実行すること。
ユーザーの未コミットファイルを変更・削除・コミットしないこと。
完了時は変更ファイル、テスト結果、未実装、既知リスク、Mac確認項目を報告して停止すること。
```

## 作業順

### Step 0: Phase 3のMac確認

コード変更なし。先に実操作を確認する。

1. editorへ移行済みプロジェクトを開く
2. 画像の配置・差し替え・hide・crop・範囲ドラッグを確認
3. 大演出の追加・内容変更・範囲ドラッグ・変化点追加/移動/削除を確認
4. 保存して更新後、再読込して状態が残ることを確認
5. 大演出中は画像バーが抑制表示され、演出終了後に有効画像へ戻ることを確認

失敗した場合はPhase 4へ進まず、下の「不具合修正テンプレート」を使う。

### Step 1: Phase 4短期対策の調査

この段階では実装しない。

```text
Phase 4短期対策の実装前調査だけ行ってください。コード変更・コミットは不要です。

目的:
replace/layout大演出が章境界まで継続している場合、生画像がページめくりの裏へ露出する問題を、
該当する章切替だけクロスフェードへ変更して解消する。

DialogueVideo.tsxで以下を特定してください:
- 章境界とページめくりの判定経路
- 境界直前・直後に有効なtopic/vizの判定方法
- panel/compare（replace/layout）とquiz/stat/callouts（overlay）の区別
- transition中に生画像が参照される箇所
- 最小変更でクロスフェードへ分岐できる位置

実装案、変更範囲、既存挙動への影響、テスト可能な純関数、Mac確認項目を提示して停止してください。
VisualSurface共通化には進まないでください。
```

判断基準: `DialogueVideo.tsx`の局所的な分岐で済まない場合は、短期対策を中止する。

### Step 2: Phase 4短期クロスフェード

Step 1が局所変更で実現可能な場合のみ実行する。

```text
承認した調査案どおり、Phase 4短期対策だけ実装してください。

要件:
- replace/layout（現行ではpanel/compare）が章境界まで有効な場合だけクロスフェード
- それ以外の章切替と既存ページめくりは変更しない
- overlay（quiz/stat/callouts）を誤って対象にしない
- meta.json形式、Python生成、editorモデルを変更しない
- 判定を純関数へ分離し、境界あり/なし、各演出種別、章頭/章末、空値をテストする
- Remotionの型チェックと既存テストを実行する
- 1コミットで停止し、VisualSurface本実装へ進まない
```

Mac確認:

1. panelが章末まで続く境界で、生画像が一瞬露出せずクロスフェードする
2. compareでも同じ
3. quiz/stat/calloutsおよび演出なしでは従来のページめくりが維持される
4. 章境界の前後1秒、停止・シーク・再生でちらつかない
5. 通常レンダリングした動画でもプレビューと一致する

1件でも失敗したらコミットを積み増して場当たり修正せず、原因を整理させる。

### Step 3: 安定化と文書更新

短期対策のMac確認後に実施する。

```text
Phase 1-4短期対策の安定化だけ行ってください。新機能は追加しません。

- docs/architecture.md、docs/DEVELOPER_GUIDE.md、docs/review-tool-spec.md、
  docs/editor-model-refactor-plan.mdを現在の実装へ合わせる
- authority、手動移行、バックアップ、legacy互換、画像/大演出の正データ、テスト方法を記載
- 古い章単位UIを現行UIとして説明している記述を修正
- 実行コマンドとMac手動確認表を1か所へまとめる
- コード変更は文書との明白な齟齬修正に限定し、必要なら先に報告する
- 文書のみの独立コミットで停止する
```

### Step 4: VisualSurface本実装の判断

自動着手禁止。短期クロスフェードで実害が解消しているなら延期する。

着手条件:

- クロスフェードでは表現上不足している
- Macで繰り返しレンダリング検証できる時間がある
- `DialogueVideo.tsx`の共通化を独立段階へ分けた設計ができている
- 現行動画の比較用サンプルを保存済み

進める場合も「VisualSurface抽出（挙動不変）」と「ページめくり対象変更」を別コミットにする。

## Claudeの完了報告に必要な項目

- コミットID
- 変更ファイルと変更理由
- 全テストのコマンドと件数
- 新規テストが防ぐ具体的な再発
- legacy/editor、保存/再読込、meta/Remotionへの影響
- 未実装と既知リスク
- Macで確認する操作を最大5件

「全件green」だけでテスト名や対象経路が無い報告は完了扱いにしない。
