# ストーリーエディタ UI 議論用ブリーフ

ChatGPT 等と UI のモック画像を作るための資料。**現状の機能・データ構造・今後の拡張性・制約**をまとめる。これを渡してレイアウト/モックを詰める。

---

## 1. これは何か（コンセプト）

ずんだもん・四国めたん等の立ち絵による「ストーリー調の会話劇動画」を作る**ローカルWeb編集ツール**。
台本（セリフの並び）と演出を画面で編集 → 音声生成 → 動画書き出し、までを1つのツールで回す。

- 動画描画は Remotion（React）。エディタはその入力 JSON を編集する GUI。
- **ローカル完結・無料・従量課金なし**が絶対条件（個人開発）。
- 既存の小ツール（`scene_editor.py` シーン配置編集 / `review_server.py` レビュー画面）と同じ作り＝**Python標準ライブラリの http.server ＋ 静的HTML ＋ JSON API ＋ Remotion Player 埋め込み**。

### 運用フロー（現状の手作業を画面化する）
```
台本を編集（話者/セリフ/シーン/表情/演出/インサート）
   ↓ 保存（story-01.json）
音声生成（VOICEVOX）← 実尺(start/end)と字幕(sentences)が自動で入る
   ↓
プレビュー / 書き出し（Remotion render → mp4）
```
※ start/end（各セリフの開始/終了秒）は音声生成で自動付与。**人間は編集しない**。

## 2. 全体構成（現状想定）

**1つのタブ式ツール**。上部にツールバー、下にタブ切り替え。

```
┌────────────────────────────────────────────────────────────┐
│ ストーリーエディタ  [第1話 ▼]      [💾保存][🔊音声生成][🎬書き出し] [状態] │ ツールバー
├────────────────────────────────────────────────────────────┤
│ [ 台本 ] [ シーン ] [ 素材 ]                                   │ タブ
├──────────────┬───────────────────┬─────────────────────┤
│ 台本一覧       │   プレビュー(Player)   │  詳細：選択ターン        │
│ (turnの並び)   │  (選択ターンに同期)    │  (選択中セリフの全項目)   │
│ 追加/削除/      │                     │                       │
│ 並べ替え        │  ▶再生・シーク        │  話者/セリフ/シーン/表情  │
│               │                     │  /演出フラグ/インサート   │
└──────────────┴───────────────────┴─────────────────────┘
```

### タブ
- **台本（Script）**: 上図の3ペイン。中核。
- **シーン（Scene）**: 場面（背景・立ち位置・モブ配置）の管理。※既存 scene_editor の機能をここに統合 or 連携。
- **素材（Assets）**: 立ち絵・背景・モブ・BGM/SE 等の管理（将来）。

---

## 3. 台本タブ（中核）の詳細

### 3-1. 台本一覧（左ペイン）
- 1行 = 1セリフ（turn）。番号・話者・セリフ先頭・シーン・演出アイコンを表示。
- 操作: 行クリックで選択、**追加 / 削除 / 並べ替え（↑↓ or ドラッグ）**。
- 場面（scene）の切り替わりが分かるよう区切り表示があると良い。

### 3-2. プレビュー（中央ペイン）
- **Remotion Player を埋め込み**、選択ターンの時刻にシークして実物を表示。
- 再生・一時停止・シークバー。編集が即反映されると理想（最低でも保存後に反映）。

### 3-3. 詳細パネル（右ペイン）＝ 1セリフの全項目
編集対象（＝turn の中身）:

| 項目 | 内容 | UI案 |
|---|---|---|
| 話者 speaker | zundamon / metan / 営業 / 部長 / AI など | ドロップダウン |
| セリフ text | 読み上げ＆字幕の本文 | テキスト入力（英字には読み仮名付与の注意書き） |
| シーン scene | 登録済みシーンから選択（office/home/server_room/rooftop…） | ドロップダウン（未登録名も許容） |
| 表情 expression | normal / happy / surprise / trouble / panic | ドロップダウン |
| 登場 enter[] | このセリフで登場させるキャラ | マルチ選択 |
| 退場 exit[] / exitDir | このセリフ後に退場するキャラ＋方向(left/right) | マルチ選択＋方向 |
| 演出 emphasis | 話者へズーム（強調） | トグル |
| 演出 shake | カメラ揺れ（緊迫） | トグル |
| 回想 flashback | 回想区間フラグ（白ディゾルブ＋彩度ダウン＋グレイン） | トグル |
| テロップ telop | 境界に出す文字（例「― 前日 ―」） | テキスト |
| 間 pause | このセリフ後の無音秒（音声生成で反映） | 数値 |
| インサート insert | PC画面/チャットの全画面表示（後述） | 種別選択＋内容編集 |

### 3-4. インサート（PC画面/チャット）編集
`insert` は1ターンに付く全画面オーバーレイ。種別ごとに内容が違う:

- **warning（システム警告）**: title, text
- **ok（正常）**: text
- **chat（AIチャット）**: user(質問1件), ai[](返答複数), highlight(強調するai番号)
- **teamchat（社内チャット/Slack風）**: channel, messages[{ from(話者ID→顔アイコン自動), text, highlight }]

UI: 種別ドロップダウン → 種別に応じた入力欄を出す。teamchat はメッセージの行追加/編集。
（補足：チャット/AIチャットを出しているターンは吹き出しを自動で抑制する＝内容重複回避）

## 4. シーンタブの詳細（既存 scene_editor の機能）

「場面（scene）」を事前登録し、台本の `scene` 名から参照する。
- **背景** bg（＋任意で前景 front＝机等でキャラ下半身を隠す layered）
- **立ち位置 anchors**（正規化座標, ドラッグ編集）＋ **cast**（どのキャラをどの位置に）
- **画角 figure**（bust=バストアップ / full=全身）, **拡大 scale**, **カメラ camera**（static / slow-zoom）, **場面転換 transition**
- **モブ配置 mobs**: モブ別に位置・サイズを D&D 編集、**表示ON/OFFトグル**（OFF=立ち絵を出さずチャット/音声のみ）

※ 現状この機能は単体ツール `scene_editor.py` に実装済み。統合するか、タブから起動/連携するかは要検討（後述の論点）。

## 5. データモデル（UIが読み書きする実体）

UIの各項目は下記 JSON に1:1で対応する。デザイン時の参考。

### 台本: `story-01.json`
```json
{
  "title": "AIが大丈夫って言ったのだ",
  "audio": "story-01.mp3",
  "script": [
    {
      "id": "turn-0004",
      "speaker": "zundamon",
      "text": "ど、どどど、どうすればいいのだ〜！？",
      "scene": "office",
      "expression": "panic",
      "enter": ["zundamon", "metan"],
      "exit": ["metan"], "exitDir": "right",
      "emphasis": true, "shake": true, "flashback": false,
      "telop": "― 前日 ―", "pause": 1.0,
      "insert": { "kind": "teamchat", "channel": "障害対応",
        "messages": [{ "from": "営業", "text": "サイト、つながらないです！", "highlight": true }] },
      "start": 7.5, "end": 10.4,           // ← 音声生成で自動付与（編集対象外）
      "sentences": [ /* 字幕単位・自動 */ ]
    }
  ]
}
```

### シーンライブラリ: `story-scenes.json`
```json
{
  "scenes": {
    "office": {
      "label": "オフィス", "bg": "background/office.png", "front": "background/office_front.png",
      "figure": "full", "camera": "slow-zoom", "scale": 1.05, "transition": "fade-black",
      "cast": { "zundamon": "right", "metan": "left" },
      "anchors": { "left": {"x":0.4,"y":0.6}, "right": {"x":0.72,"y":0.6} },
      "mobs": { "営業": {"x":0.5,"y":0.99,"scale":0.85,"hidden":true} }
    }
  }
}
```

### 登場人物
- **主役**: zundamon / metan（パーツ立ち絵＝口パク・表情・退場あり）。将来追加可能。
- **モブ**: 営業 / 部長（いらすとや風の1枚絵・通常/取り乱しの2状態・口パク無し）。チャット顔アイコンあり。
- **AI**: 画面に立ち絵は出さず、音声＋チャット(chat)で登場。

## 6. 今後の拡張性（モック設計で考慮してほしい）

- **複数エピソード**: story-01, story-02 … を切り替え（ツールバーのエピソード選択）。プロジェクト管理（新規作成/複製）。
- **キャラ追加**: 主役・モブを増やせる前提（話者ドロップダウン・素材タブ）。
- **インサート種別の追加**: warning/ok/chat/teamchat 以外（メール画面、ターミナル、地図、画像インサート等）が増える余地。種別ドロップダウンが拡張可能な作りに。
- **BGM / SE**: 区間ごとのBGM、効果音（アラート音等）の付与（タイムライン/タブ）。
- **タイムライン表示**: 現状は「セリフの並び（リスト）」だが、将来は時間軸タイムライン（尺・BGM・SE・演出を帯で）に発展する余地。リストとタイムラインの両立 or 切替。
- **台本生成（AI下書き）**: ストーリーのあらすじから台本ドラフトを生成 → 手直し、というフローを将来差し込む（無料枠のLLM想定）。
- **シーン未登録の扱い**: 台本が参照するシーンが未作成でもエラーで止めず「未登録」を可視化し、後から作成して紐付け。
- **モバイル/タブレットでの確認**: 外出先で軽く確認・微修正できると嬉しい（必須ではない）。

## 7. デザイン方針・制約

- **フラットでミニマル・モダン**。ダークテーマ（既存 scene_editor が `#0f1117` 系のダーク）。
- 情報密度は高め（編集ツールなので）だが、3ペインで役割を分離して迷わせない。
- **WYSIWYG**: プレビューと実書き出しが一致すること（座標・サイズの計算をrenderと合わせる思想）。
- 技術制約: ローカルの http.server（標準ライブラリ）＋静的HTML/JS（フレームワーク無しでも可）。Remotion Player は npm。**外部送信・課金なし**。
- 1人開発・初期は速度優先（過剰な抽象化はしない）。まず台本編集→保存が回ることを最優先。

## 8. 議論したい論点（ChatGPTに投げる用）

1. **レイアウト**: 3ペイン（一覧/プレビュー/詳細）で良いか。プレビューを上部固定にして下に一覧＋詳細、等の別案は？
2. **台本一覧の表現**: 単純リスト か、簡易タイムライン（尺を反映した帯）か。現状は尺が音声生成後にしか出ない点に注意。
3. **詳細パネルの情報量**: 項目が多い（表情/演出フラグ/インサート）。アコーディオンやセクション分けでどう整理するか。
4. **シーン編集の統合 vs 分離**: 同じツールのタブに入れるか、別ツール（既存 scene_editor）を起動/リンクするか。
5. **インサート編集UI**: 種別が増える前提で、どう拡張しやすく見せるか。
6. **音声生成・書き出しの動線**: ボタン位置、進捗表示、未保存時の扱い。
