# レビューツール詳細UI 改善実装指示書

Claude Codeが `/story` の詳細編集UIを段階的に改善するための作業指示書。

## 1. 目的

メインレイアウトは維持し、以下の詳細操作を安全で分かりやすくする。

- セリフ編集
- テキスト範囲演出・小演出編集
- 大演出編集
- 画像検索・画像補正
- 章編集

動画生成機能の追加やレンダリング表現の変更は目的に含めない。

## 2. 最初に読むもの

作業開始前に以下を読み、現在の実装と保存境界を把握すること。

1. `AGENTS.md`
2. `docs/architecture.md`
3. `docs/DEVELOPER_GUIDE.md`
4. `docs/review-tool-spec.md`
5. `review_story_page.html`
6. `review_server.py` の `/api/script`、`/api/options`、`/api/delete-cut` 周辺

主な変更対象は `review_story_page.html` とする。必要性がない限り、Python APIを変更しない。

## 3. 変更禁止範囲

- `meta.json` の形式を変更しない。
- `DialogueVideo.tsx` を含むRemotion側の描画ロジックを変更しない。
- VOICEVOX、台本生成、画像取得パイプラインを変更しない。
- `script.json`、`review.json`、画像、音声などの制作データをコミットしない。
- 新しいフレームワークや外部依存を追加しない。
- `review_server.py` や `review_story_page.html` をフルリライトしない。
- 既存の未コミット変更を上書き、削除、巻き戻ししない。

## 4. 作業前の保護

1. `git status --short` で既存変更を確認する。
2. 制作データの変更は作業対象から除外する。
3. UIコードに既存の未コミット変更がある場合は内容を読み、共存できない場合だけユーザーへ確認する。
4. 1フェーズごとに動作確認し、内容を分けてコミットする。

## 5. 実装優先順位

### フェーズ1: 編集事故の防止

#### 5.1 セリフ編集の確定と取消

対象: `lineRow()`、`startEditLine()`

- ダブルクリックに加え、選択行に編集ボタンを表示する。
- 編集開始時の文字列を保持する。
- `Escape` で編集前の文字列へ戻して終了する。
- `Cmd+Enter` / `Ctrl+Enter` で確定する。
- フォーカスが外れた場合の挙動を統一し、ユーザーに分からないまま確定・取消しない。
- 編集終了時に左ペインのスクロール位置と選択行を維持する。
- グローバル保存が必要な変更は、従来どおり未保存状態として明示する。

受け入れ条件:

- 入力後に `Escape` を押すと本文とテキスト演出が編集前の状態へ戻る。
- 確定後もフォーカスや左ペインが先頭へ飛ばない。
- 空文字、複数行、日本語入力中でもショートカットが誤作動しない。

#### 5.2 テキスト演出の保護

対象: `startEditLine()`、`addTextEffect()`、`renderTextEffectEditor()`

- 同じ単語が複数存在しても、既存演出を先頭一致へ勝手に移動しない。
- セリフ編集で演出対象を一意に維持できない場合は、黙って削除・移動せず「再選択が必要」と表示する。
- 新しい範囲演出の追加時、重なった既存演出を無警告で削除しない。
- 現行のRemotion描画が重複演出に対応している場合は重複を保持する。
- 対応していない場合は既存演出を残したまま、追加できない理由を表示する。
- 適用済み演出を一覧から選択し、対象範囲と種類を確認できるようにする。

受け入れ条件:

- 同じ語句が2回あるセリフを編集しても、演出対象が別の語句へ移動しない。
- 重複範囲を追加しても既存演出が無警告で消えない。
- 既存の `textEffects` データとの後方互換性を保つ。

#### 5.3 削除操作の分離

- セリフの「削除」を「分割」から視覚的に離すか、その他メニューへ移す。
- 「演出を削除」は通常設定から区切った危険操作エリアへ移す。
- 画像カット削除は即時保存されることと、後続カットが再採番されることを確認画面に明記する。
- 確認ダイアログを増やしすぎず、取り返しのつかない操作だけを対象にする。

### フェーズ2: 右ペインの情報設計

#### 5.4 共通構造

- 右ペイン上部のタブと選択中セリフ情報をスクロール中も確認できるようにする。
- 現在の対象を `章 / 話者 / セリフ番号` で短く表示する。
- 「広げる」は維持し、通常幅でも主要操作が完結する構造を優先する。
- ペイン再描画時のスクロール位置保持を壊さない。

#### 5.5 小演出タブ

タブ内を次の3セクションへ分ける。

1. テキスト範囲演出
2. テロップ
3. リアクション

- セクションは見出しと適用件数を持たせる。
- テキスト選択欄には「文字をドラッグして選択」と常時分かる表示を置く。
- 色入力には画面上のラベルを付け、ツールチップだけに依存しない。
- リアクションの `×` は「解除」など、結果が分かる名称にする。

#### 5.6 大演出タブ

- 現在の「左レール」を前提にした古い範囲調整説明を削除する。
- タイムラインの左右端ドラッグで範囲調整できることを短く案内する。
- 設定を「内容」「表示タイミング」「デザイン」に分ける。
- 色、透明度、文字サイズなど低頻度の項目は「デザイン」に折り畳む。
- 演出追加セレクトには、各演出の用途が分かる短い補足を付ける。
- 既存の演出種別と保存データ形式は変更しない。

#### 5.7 画像タブ

- 画像検索の英語入力と日本語入力を、可能なら単一入力欄と取得操作へ整理する。
- API仕様上統合が難しい場合は、通常使用する日本語検索を先に置き、英語検索を詳細設定へ移す。
- 「基本」「補正」「差し替え・出典」の折り畳みセクションへ整理する。
- 画像補正が即時保存であることを補正セクション内に表示する。
- `cover`、`contain`、`ambient`、`subject` は日本語の説明を主表示にする。
- 1280x720で、画像選択と検索がスクロールなし、基本補正が少ないスクロールで操作できることを目標にする。

#### 5.8 章タブ

- 章番号と現在の章タイトルをヘッダーに表示する。
- タイトルと要約の用途を短く説明する。
- 章の編集内容もグローバル保存が必要だと分かるようにする。

### フェーズ3: 発見性とレイアウト調整

- 狭い左ペインで「章を編集」が縦書き状に潰れないようにする。
- 必要ならアイコンボタンまたは章メニューへ変更する。
- 編集・分割・削除の操作は選択行だけに表示し、本文を圧迫しないようにする。
- タブレット向けの全面再設計は行わないが、1280x720で主要操作が成立することを確認する。
- 既存の台本モード50:50、Remotionプレビュー、下段タイムラインを維持する。

## 6. 保存モデルの扱い

現状は保存方式が2種類ある。

- セリフ・章・演出: `DATA` を更新し、グローバル保存で `/api/script`
- 画像補正・差し替え・カット削除: `/api/options` などで即時保存

今回、APIを大きく変更して完全統一しない。まずUI上で次を明確にする。

- 未保存の変更には従来の保存表示を維持する。
- 即時保存する画面には「変更は自動保存されます」と表示する。
- 即時保存操作の失敗時は、成功したように見せずエラーを表示する。
- 保存方式を変更する必要が生じた場合は、そのフェーズを止めて影響範囲を報告する。

## 7. 検証

外部APIや従量課金サービスを使わず、ローカルだけで確認する。

```bash
.venv/bin/python review_server.py --dir docs/story --port 8765
```

最低限、以下を手動確認する。

1. 1280x720と1440x900で `/story` を開く。
2. 台本モードと演出モードを切り替える。
3. セリフを編集、確定、取消、分割する。
4. 同じ単語を2回含むセリフでテキスト演出を確認する。
5. 小演出を追加・削除し、タイムラインとプレビューの同期を確認する。
6. 大演出の設定と範囲調整を行う。
7. 画像補正画面を開き、即時保存表示とエラー表示を確認する。
8. 右ペインやタイムラインが操作後に先頭へ戻らないことを確認する。
9. 保存後に再読み込みし、編集内容が維持されることを確認する。

既存のローカルテストも実行する。

```bash
.venv/bin/python test_story_script.py
.venv/bin/python test_story_meta.py
```

## 8. コミット単位

以下を目安に分ける。

1. `fix: make dialogue and text effect editing safe`
2. `refactor: organize review detail panels`
3. `fix: improve detail editor usability on laptop screens`

各コミットには関係するコードとドキュメントだけを含める。制作データや既存の無関係な変更を含めない。

実装前に関連コードと保存モデルを確認し、指示書の受け入れ条件を満たす最小変更にしてください。ローカルで動作確認と既存テストを行い、フェーズ1の変更だけをコミットしてください。完了報告では、変更ファイル、確認した操作、テスト結果、残課題を簡潔に示してください。
