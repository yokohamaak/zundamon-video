# ストーリー調 会話劇動画エディタ 仕様

> 既存の「ずんだもん掛け合い解説動画」ツールとは**別の新規ツール**。既存ツールはそのまま残す。
> 本書は 2026-06-26 の整理で確定した内容を上部に集約し、初期の検討メモを後半に資料として残す。

---

## 1. 目的

ずんだもん達が**全画面で会話しながら物語が進む**動画を、専用エディタで素早く作る。

- どじっ子SE（ずんだもん）が「やらかす→焦る→困る→解決」のようなストーリー仕立て
- 自由配置ではなくプリセット／ライブラリベースで素早く組む
- 背景・キャラ・構図を事前準備した素材から選んで組み立てる

参考動画: `docs/reference/videoplayback.mp4`（睡眠/SASテーマ）, `docs/reference/sample1.mp4`

---

## 2. 全体方針（確定）

### 2.1 既存ツールとの関係

- 既存の掛け合い解説動画ツールは**触らない**。これは新規ツール。
- 既存資産は「**流用が楽なら使う、楽でなければ新規**」。きれいな統合は目的ではなく「やりたいことができる」が優先。
- 「使う」の2種類を区別する:
  - **ワークフロー流用**: 既存プログラムを動かして出力を貰う
  - **パーツ流用**: 既存コードを新ツールに取り込む

### 2.2 1ツールで完結

台本生成〜描画まで1つのツールで完結させる。理由: 既存ツールの台本生成は「実は〇〇が〇〇」固定プロンプト（旧trivia路線）でストーリー調に合わないため、台本部分は新規が必要。

### 2.3 形態（既存レビューツール review_server.py に倣う）

- ローカル専用Webアプリ（**標準ライブラリ/http.server のみ**）・ブラウザUI
- **静的フロント ＋ JSON API** 分離（相対URLで `/api/*`＝将来 Cloudflare 等へ移行容易）
- **Remotion Player を埋め込んでプレビュー**
- 承認チェックポイント式の半自動（human-in-the-loop）
- **ローカル完結・無料枠のみ**（従量課金・情報漏洩リスクのある実装は禁止）

## 3. 制作フロー（end-to-end）

```
① 台本生成（新規）        AIたたき台。ストーリー用プロンプト（既存の固定プロンプトは使わない）
        ↓
② セリフ編集（新規）        手直しUI。話者・本文・scene・expression を直す
        ↓
③ 音声化（VOICEVOX等・流用） turn の start/end が確定
        ↓
④ 絵付け（新規）           scene 区切り・背景・キャラ配置・構図
        ↓
⑤ 吹き出し（新規）         話者キャラ近くに小ボックス
        ↓
⑥ 描画（Remotion・流用）   back→キャラ→front の重ね・ズーム/パン・立ち絵Avatar
```

- **新規で作る**: ①②④⑤
- **流用**: ③（音声化）⑥（描画・立ち絵Avatar）
- **台本＝AIたたき台→手直し**の両方式（生成プロンプトは新規・編集UIが要る）

## 4. データモデル（確定）

### 4.1 台本スキーマ

AIが出力し、セリフ編集UIで手直しし、音声化で start/end が埋まる。

```jsonc
{
  "title": "なぜ朝起きても疲れが取れないのか",
  "script": [
    {
      "id": "turn-0001",
      "speaker": "zundamon",       // 話者（キャラID）
      "text": "もう朝なのに体が重いっす…",
      "scene": "bedroom",           // ★AI指定：シーン名（登録ライブラリ参照／未登録なら後で紐づけ）
      "expression": "trouble",      // ★AI指定：表情（§4.3）
      "enter": ["zundamon"],        // 登場キャラ（緩い・退場は手動）
      "face": { "zundamon": "right" }, // 任意：向きの明示指定。省略時は立ち位置から自動(中央向き)
      "start": null,                // ←音声化で自動付与（AIは書かない）
      "end": null
    },
    {
      "id": "turn-0002",
      "speaker": "metan",
      "text": "また寝てたの？",
      "scene": "bedroom",           // 同じ→ツールが同一sceneにまとめる
      "expression": "normal",
      "enter": ["metan"],           // ここで めたん 登場（ずんだもんは継続表示）
      "start": null,
      "end": null
    }
  ]
}
```

設計上のルール:

- **scene は毎turnに書く**（変化点だけでなく）。AIは状態を持たず自己完結。ツールが「sceneが変わった所」を検出して scene 区切りにする。
- **start/end はAIが書かない**。尺は音声化（③）で決まる。
- **scene 名は登録ライブラリのキー**。未登録なら「未登録」を検出表示し、後からシーンを作って紐づける（エラーで止めない＝既存エディタの orphaned/保留と同思想）。
- **enter（登場）は緩い指定**。一度出たら手動で消すまで居続ける。退場は手動。ズームで1人だけ表示等は構図側で扱い厳密でなくてよい。
- **face（向き）は任意の上書き**。省略時は立ち位置から自動で中央を向く（左配置→右向き／右配置→左向き）。立ち絵素材は「画面左向き」が素なので、右を向かせるときだけ左右反転する。
- turn の基本構造（id/speaker/text/start/end）は既存ツールから流用。

### 4.2 シーン（2層構造）

シーンには「再利用するライブラリ登録」と「動画内の区間インスタンス」の2層がある。

**(a) シーンライブラリ登録（再利用・シーンエディタで作る）**

| 登録項目 | 内容 |
|---|---|
| シーン名 | `bedroom` / `office` / `home` 等（台本の scene が参照するキー） |
| 背景画像 | flat（1枚）。任意で front レイヤー（透過PNG）を足すと layered |
| 立ち位置（anchor） | 主役キャラを置く名前付き位置。主役はしっかり／モブは緩く |
| 既定の構図・カメラ | `shot`（§4.5）／`cameraMotion` の既定値（例: office→solo＋slow-zoom） |

**(b) 動画内の scene 区間（その動画固有・ツールが台本から導出）**

- 台本の `scene` 名が連続する範囲を1区間にまとめる（§4.1）。
- ライブラリのシーンを参照し、**構図・カメラはライブラリ既定を引き継ぎ、絵付けエディタで上書き可**。
- 登場キャラは台本の `enter` 由来。
- **AIは台本で scene 名と表情しか指定しない**ので、構図・カメラはAI出力に含めず (a)(b) で決める＝AI出力をシンプルに保つ。

背景方式:

- **flat**: 1枚背景＋ゆっくりズーム/パン
- **layered**: `back → キャラ → front` の重ね順。front（透過PNG）で「机の後ろに立つ」構図（参考動画 42–46 秒のオフィス机が実例）。
  - 描画は軽い（z順だけ）。負担は front 素材の準備（切り抜き）。front 無し＝flat と同じ。
- **parallax**: MVPでは**なし**。ただし深度（`make_depth.py`）は「front レイヤーを半自動で切り出す下ごしらえツール」に転用候補（運用後・仕上げは手調整前提）。

### 4.3 キャラクター

| 種別 | 表現 | 配置 |
|---|---|---|
| 主役（ずんだもん・めたん等） | パーツ分け立ち絵：口パク・表情・ポーズ変化 | シーンの anchor に配置 |
| モブ（いらすとや風等） | **1枚絵で固定**（表情・口パクなし） | 緩い配置。最低限「登場する」が分かればよい |

立ち絵は既存 `video/src/Avatar.tsx`（パーツ重ね方式）をほぼ流用。

### 4.4 吹き出し（確定）

- **位置**: 話者キャラの**足元あたりに自動配置**。話者の画面上x位置を基準にし、背景固有座標には依存させない。
- **タイミング**: turn の `start`〜`end` で表示。**同時表示は1つ**（前のセリフは残さない）。音声と同期。
- **長文の出し方**: 1 turn のテキストを**文単位（既存 `sentences[]` 流用）で小出し**。turn 内で文が変わるたびに吹き出しを差し替え。台本は1 turn＝1発言のまま、表示だけ文で割る。
- **デザイン**: 小型ボックス・名前ラベルなし・1〜2行・約24文字目安。
- 資料オーバーレイと衝突する場合は自動でずらす／無理なら代替位置（Phase 5 で対応）。

#### 表情語彙（実物を見て再調整前提でいったん確定）

| 表情 | 書き出す合成パーツ | 元レイヤー（坂本アヒル版V3.2） |
|---|---|---|
| `normal` | eye_open | 基本眉＋基本目 |
| `happy` | eye_smile | 基本眉＋にっこり/^^ |
| `surprise` | eye_surprise ＋ fx_surprise | 上がり眉＋見開き目＋びっくり記号 |
| `trouble`（困り） | eye_trouble | 困り眉＋なごみ目＋かげり |
| `panic`（焦り） | eye_panic ＋ fx_sweat | 困り眉＋見開き/基本目＋青ざめ／汗 |

立ち絵素材: `docs/reference/ずんだもん立ち絵素材V3.2/*.psd`（製作: 坂本アヒル）。
公式（zunko.jp/guideline）準拠で改変・商用OK・クレジット任意。
眉・目・口・顔色（赤面/青ざめ/かげり）・記号（汗/涙）・腕/枝豆ポーズが揃い、困り/焦り/怒り/泣きまで作成可能。

実装方針:

- 既存 Avatar は眉が独立レイヤーでなく `eye_*` に表情を込める方式。
- **案A（推奨・MVP）**: 表情ごとに「眉＋目＋顔色」を1枚に合成して `eye_trouble.png` 等として書き出す。既存方式のまま種類を増やすだけ。口はリップシンク用に分離維持。
- 案B: 眉・顔色・記号レイヤーを Avatar に追加して動的合成（柔軟・コード増）。
- 書き出しは PSDTool（oov.github.io/psdtool・無料ブラウザツール・課金なし）。`fx_sweat` は Avatar.tsx 側で対応済（画像が無いだけ）。

### 4.5 構図・カメラ（確定・最小セット）

参考動画は「1人上半身」「2人並び」「分割2画面」、カメラは静止かゆっくり寄りで成立していた。過剰な細分を避け最小に絞る。

**shotPreset（構図）**

| preset | 内容 | MVP |
|---|---|---|
| `solo` | 1人。上半身基準。左/中央/右の寄せを指定 | ✓ |
| `duo` | 2人。左右に配置 | ✓ |
| `split` | 分割2画面（左右別背景・別キャラ） | Phase 5（描画が別物なので後） |

初期案の `solo-full/upper/close`・`duo-left-focus` 等の細分、`trio-wide`・`with-overlay-center` は当面不要。`solo`/`duo` に「寄せ」と「ズーム量」を持たせれば足りる。

**cameraMotion（カメラ）**

| motion | 内容 | MVP |
|---|---|---|
| `static` | 静止 | ✓ |
| `slow-zoom` | ゆっくり寄る（既存 Ken Burns 流用） | ✓ |

`slow-zoom-out`・`wide-to-focus` は後回し。

**持ち主**: 構図・カメラは **scene が持つ**（背景が変わる＝構図が変わる単位）。scene に `shot`（solo/duo＋寄せ・ズーム量）と `cameraMotion`（static/slow-zoom）を持たせる。

## 5. 着手順序（合意）

「まず動くものを作る・ローカルで回す」に沿い、**描画を固定入力で先に動かす**。

```
Phase 1【絵を出す】★最初
  手書きの台本JSON（§4.1）＋仮の背景/立ち絵で
  Remotion描画（back→キャラ→front・立ち絵Avatar・吹き出し）を動かす → 見た目が確定

Phase 2【シーンエディタ】 シーン登録UI（背景＋anchor＋名前）。scene名と紐付け・未登録検出
Phase 3【セリフ編集＋音声化】 セリフ編集UI ＋ VOICEVOX流用で start/end 確定
Phase 4【AI台本生成】 ストーリー用プロンプトでたたき台（使えるシーン一覧を渡す）
Phase 5【仕上げ】 構図/カメラのプリセット、吹き出し衝突回避、表情パーツ書き出し(困り/焦り)
```

理由: Phase 1 で見た目が確定すると以降の判断（吹き出し位置・構図）を実物を見ながらできる。重い依存（AI・TTS）は後ろへ。

# （資料）初期検討メモ

> 以下は最初期のたたき台。確定版は上部 §1–4 を参照。用語や背景アセットJSONの素案として残す。

## 基本方針（初期案）

- 編集の基本単位は `scene`、セリフは `turn`
- 背景はライブラリ登録して再利用する
- 自由座標編集を避け、`anchor` / `shotPreset` / `cameraMotion` の組み合わせで制御する

## 用語

- **turn**: 1発話単位のセリフデータ。話者・本文・音声タイミング。
- **scene**: ある区間の画面構成（背景・登場キャラ・構図・カメラ・資料表示）。`turnStartId`/`turnEndId` で範囲を持つ。
- **background asset**: 背景ライブラリに登録した再利用素材。表示方式・アンカー情報を持つ。
- **anchor**: 背景ごとの配置基準点。キャラ立ち位置や資料位置に使う。
- **shotPreset**: 構図プリセット（1人全身・2人引き・資料つき等）。
- **cameraMotion**: カメラ遷移プリセット（静止・寄る・引く等）。

## 背景アセット仕様案（初期）

```jsonc
{
  "id": "office_desk_001",
  "label": "オフィス・デスク前",
  "mode": "layered",            // flat / parallax / layered
  "variants": {
    "day":   { "layers": { "back": "..._day_back.png",   "front": "..._day_front.png" } },
    "night": { "layers": { "back": "..._night_back.png", "front": "..._night_front.png" } }
  },
  "anchors": {
    "main":    { "x": 0.52, "y": 0.88, "label": "机前中央" },
    "left":    { "x": 0.34, "y": 0.90, "label": "机前左" },
    "right":   { "x": 0.72, "y": 0.90, "label": "机前右" },
    "overlay": { "x": 0.08, "y": 0.10, "w": 0.42, "h": 0.34, "label": "左上資料枠" }
  }
}
```

## shotPreset 案（初期）

- 1人用: `solo-full` / `solo-upper` / `solo-close`
- 2人用: `duo-wide` / `duo-left-focus` / `duo-right-focus`
- 3人用: `trio-wide`
- 資料系: `with-overlay-center`

## cameraMotion 案（初期）

- `static` / `slow-zoom-in` / `slow-zoom-out` / `wide-to-focus`

## セリフUI仕様案（初期）

- 話者キャラ近くに小型ボックスで表示。同時表示は1つ。名前ラベル基本なし。1〜2行。
- 目安は約24文字／吹き出し。長いセリフは turn 分割。
- 資料オーバーレイと衝突する場合は自動でずらす／無理なら代替位置へ。
