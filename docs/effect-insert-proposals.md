# 演出 / インサート追加候補一覧

目的:
- 今ある `warning / ok / chat / teamchat / mailer` や `emphasis / shake / flashback / telop` を前提に、
  次に足すと効きそうな演出・インサート候補を整理する
- 後から見返しやすいよう、`汎用度 / 優先度 / 実装コスト` を同じ尺度で並べる

前提:
- 世界観は `株式会社ずんだシステムズ` のIT障害対応ストーリー
- 優先度は「シリーズ全体への効き方」と「今の手持ち表現の穴を埋める度合い」で評価
- 実装コストは今の `StoryVideo.tsx` / `story_editor.html` / `story_editor.py` / プロンプト更新込みの概算

評価の目安:
- 汎用度: `高 / 中 / 低`
- 優先度: `A(かなり欲しい) / B(あると強い) / C(余裕があれば)`
- 実装コスト: `小 / 中 / 大`

## 優先実装候補

### 1. `terminal`
- 理由: IT障害対応の「調査している感」を最も自然に足せる
- 汎用用途:
  - `curl` / `ping` / `nslookup`
  - `tail -f` 的ログ
  - `systemctl status`
  - エラー文の抜粋
- 実装メモ:
  - 最初はプレーンテキスト複数行だけでも十分
  - カラー文字やプロンプト記号は後回しでよい

### 2. `statusboard`
- 理由: 障害回の導入・中盤・復旧後まで全部で使える
- 汎用用途:
  - ステータス: `発生 / 調査中 / 復旧 / 監視中`
  - 影響範囲、優先度、担当、開始時刻
- 実装メモ:
  - 1画面カードUIで収まるので実装が軽い
  - `warning` の派生として考えやすい

### 3. `whiteboard`
- 理由: めたんの「答えではなく考え方を教える」を可視化できる
- 汎用用途:
  - 仮説一覧
  - 切り分け手順
  - 教訓3点まとめ
- 実装メモ:
  - 箇条書きだけでも成立する
  - 教育っぽさが増すわりに実装は軽い

### 4. `browser`
- 理由: 視聴者に状況を伝える力が強い
- 汎用用途:
  - `503 Service Unavailable`
  - 真っ白画面
  - ログイン不可
  - 管理画面の警告
- 実装メモ:
  - 最初は「ブラウザ風フレーム + タイトル + 本文」だけでもよい

### 5. `diff-config`
- 理由: 「良かれと思って設定を変えた」がそのまま画になる
- 汎用用途:
  - `.env`
  - `nginx.conf`
  - `docker-compose.yml`
  - JSON/YAML設定
- 実装メモ:
  - 差分色分けは最初は単純でも成立する

## 低コストで足しやすい候補

- `statusboard`
  - 新しいインサート1種として追加しやすい
- `whiteboard`
  - レイアウト自由度が高く、必要データも少ない
- `spotlight`
  - 既存のカメラ演出に重ねやすい
- `freeze-note`
  - 一時停止 + 注釈表示なので構造が単純
- `callout-arrow`
  - 他インサートの補助演出として汎用性が高い

## 世界観との相性が特に良い候補

- `terminal`
  - IT何でも屋の空気が一番出る
- `statusboard`
  - 障害対応ストーリーの導入に強い
- `ticket`
  - 社内運用感が出る
- `deploy-board`
  - 「昨日何した？」系の原因表示に強い
- `diff-config`
  - ずんだもんのやらかしを視覚化しやすい

## 補足

- まずは「汎用度が高く、データ構造が単純なインサート」を増やすのが得
- 派手な演出よりも、障害対応の文脈を補強する表示系の方がシリーズ全体に効く
- 新規追加時は `docs/new-effect-checklist.md` にある通り、
  `StoryVideo / story_editor.py / story_editor.html / 取り込み検出 / プロンプト` の更新が必要
