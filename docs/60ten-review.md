# 60点改善 — 確認・レビュー手順

「実は雑学」動画が60点だった件の改善。原因を3領域（台本・面白さ／音声・間・尺／演出・アニメ）に切り分け、
コンテナ側で詰められる修正を入れた。**効果確認は render（Mac固定）が要る**ため、この手順で検証する。

対象動画: 「実は知らないデジタルの名前の謎」（USB / Wi-Fi / Bluetooth / .com / CAPTCHA・約6.6分）

---

## 1. この回の変更（3点）

| # | 領域 | 変更 | 確信度 | 無効化/調整 |
|---|------|------|--------|-------------|
| A | 台本・面白さ | 生成プロンプトの型の単調さを解消（繋ぎ/明かし/締めを毎回変える・冒頭フック強化）。通常版＋再生成版の両方 | 高（出力に直接現れる） | プロンプト文言の差し戻し |
| B | 音声・間 | ネタ(章)の境目に追加の「間」を入れる `chapter_gap_pause`（既定0.35秒） | 中（要 render/試聴） | `config.story.yaml` で `0` にすると無効 |
| C | 演出 | 聞き手キャラのうなずき（相槌）を追加。静止による単調さを緩和 | 中（要 render） | `Avatar.tsx` の `nod` を `0` に |

- A: `src/story_script.py`（commit 0b88290）
- B: `src/tts_voicevox.py` + `config/config.story.yaml`
- C: `video/src/Avatar.tsx`

A だけでも効くが、B/C はブラインド（試聴・render前提）の仮修正。**ダメなら個別に戻せる**ように設計済み。

## 2. 確認手順（Mac）

> 前提: VOICEVOX をホスト(Mac)で起動（:50021）。`.env` に `PEXELS_API_KEY` / `PIXABAY_API_KEY`（無料登録・Wikimediaは無認証）。

### 2-1. ロジック検証（先に・無料）
```bash
python3 test_tts_voicevox.py     # chapter-gap 含む TTS 回帰
python3 test_story_script.py     # プロンプト構造
```
両方 `ALL PASS` を確認。

### 2-2. 台本だけ先に目視（再生成・画像/音声なし）
```bash
python main_story.py --script-only
```
→ `docs/story/script.json`（または出力先）を開き、**3章レビュー**（下記）で台本だけ先に判定。
NGなら render前にプロンプトを再調整 → 従量・時間の無駄を防ぐ。

### 2-3. 音声＋画像つきでフル生成
```bash
python main_story.py             # --no-images を付けない（実画像取得）
```

### 2-4. render
```bash
cd video && SRC_DIR=../docs/story npm run render
# 反復中は npm run dev（Studio/HMR）で速く確認、最終だけ render
```

## 3. レビュー手順（チェックリスト）

各項目を ○ / △ / × で。× があれば「調整の効かせ方」に従って詰める。

### 3-A. 台本・面白さ（script.json / 動画前半で判断）
- [ ] **繋ぎが毎ネタ違う**か（「ところで」が4連発していない）
- [ ] **明かしが毎回「実はね」始まりでない**か（切り出しに変化）
- [ ] **締めが毎回「つまり〇〇は〜なのだ」でない**か（要約／感想／ツッコミで変化）
- [ ] **冒頭で一番の山場をチラ見せ**して掴んでいるか（ただの挨拶始まりでない）
- [ ] 各ネタが独立して「へぇ」になる正確な事実か（嘘・誇張なし）
- [ ] 山場（最後のネタ）が一番強いか

判定の勘所: **前半3ネタで「同じパターンの繰り返し」に見えたら ×**。型が透けるのが60点の主因だった。

### 3-B. 音声・間・尺（試聴で判断）
- [ ] **ネタの切り替わりに一拍の区切り**があり、メリハリが出たか（埋もれていない）
- [ ] 間が**長すぎて間延びしていない**か（逆効果なら短く）
- [ ] 話者交代・文の境目がカクついていないか
- [ ] 総尺が長すぎ/短すぎないか（目安7分前後）

判定の勘所: 章境界の間は**入れすぎると逆にもたつく**。0.35で間延びを感じたら 0.2〜0.25、区切りが弱ければ 0.4〜0.5。

### 3-C. 演出・アニメ（render を見て判断）
- [ ] **聞き手が静止せず、相槌（うなずき）で生きて見える**か
- [ ] うなずきが**過剰/不自然でない**か（チック様に頻繁すぎない・大きすぎない）
- [ ] 発話者の口パク・感情リアクション（surprise跳ね/happyホップ）が台詞と合っているか
- [ ] zoom_punch（種明かし）/ flash（ネタ切替）が効果的に入っているか
- [ ] 画像の切替・Ken Burns がカクついていないか

判定の勘所: うなずきが気になるなら振幅 `* 5` を `* 3` に、頻度は周期 `0.4`(Hz) を `0.3` に。

## 4. 調整の効かせ方（パラメータ早見）

| 効かせたいこと | 場所 | 値 |
|----------------|------|-----|
| ネタ境界の間を増減 | `config/config.story.yaml` `chapter_gap_pause` | 既定0.35。0で無効。0.2〜0.5で調整 |
| 話者交代の間 | 同 `inter_turn_pause` | 既定0.3 |
| 文末の余韻（一文ずつの区切り感） | 同 `post_phoneme_length` | 既定0.1 |
| 喋り速度 | 同 `voice_params` 各話者 `speed` | めたん0.92 |
| 尺の目安 | 同 `target_minutes` / `topics` | 7分 / 5ネタ |
| うなずきの振幅 | `video/src/Avatar.tsx` `nod = ... * 5` | 5px。0で無効 |
| うなずきの頻度 | 同 `* 2 * 0.4`（Hz） | 0.4≈2.5秒周期 |

## 5. 次の一手（render後に判断）

新台本で再評価し、**まだ単調なら**:
- 演出: effect配分の再調整（新台本の山場に zoom_punch / flash を寄せる）
- 間: pause / 無音をさらに耳調整
- 台本: 物足りなければプロンプトに「具体例・身近な例え」「数字のインパクト」を足す方向

A（台本）が効いているかをまず最優先で判定。台本が直れば演出・音声の体感も変わるため、**台本 → 音声 → 演出 の順で確定**していく。
